# FIA 기관 API 어댑터 교체 계약

## 현재 MVP

`lib/action-adapters.ts`의 `InstitutionAdapter`가 금융기관 실행 경계다. 현재 구현체인 `SandboxInstitutionAdapter`는 실제 금액을 이동하지 않으며 사용자 승인, 금액 유효성, L4 1회 500만원 한도, 기관 실패와 취소를 결정론적으로 재현한다.

## 상용 어댑터 필수 계약

상용 구현체는 다음 조건을 동일하게 보장해야 한다.

1. 입력: 액션 ID, 멱등키, 사용자 권한, 서비스·기능, 금액, 명시적 승인 상태
2. 출력: `success | failed | cancelled`, 안정적인 결과코드, 사용자 설명, 기관명, 외부 참조번호
3. 멱등성: 같은 키는 같은 기관 결과를 반환하고 송금·주문을 다시 만들지 않음
4. 인증: 비밀번호·OTP·인증서 비밀키를 FIA에 저장하지 않고 기관 인증 화면 또는 제한 토큰 사용
5. 검증: 계좌 소유권, 상품 거래 가능 여부, AML·제재, 1회·일간 한도와 기관 가용성 확인
6. 감사: 요청·정책 판단·기관 응답·취소·보상 상태를 D1 액션과 감사로그에 함께 기록
7. 실패 안전: 시간초과 결과를 성공으로 추정하지 않고 기관 조회 API로 최종 상태 대조

## 실제 연동 대상

- 자산 조회: 승인된 마이데이터·오픈뱅킹·카드·보험·증권·가상자산 사업자 API
- 실행: 은행 송금, 증권 주문, 카드·공과금, 환전, 대출·보험 신청의 기관별 인증 API
- 시장·뉴스: 계약된 시세·공시·뉴스·정책·공공데이터 API
- 세금·규정: 국가별 공식 법령·신고 채널과 전문가 검수

실제 서비스 전에는 국가별 인허가, 전자금융·개인정보·신용정보 규제, 소비자보호, 설명의무, 기록 보존기간과 침해사고 대응을 법률·보안 전문가가 검토해야 한다.
