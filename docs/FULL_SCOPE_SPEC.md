# FIA Full-scope MVP 기능 명세 및 완료 기준

## 제품 정의

FIA는 개인·가계·개인사업자·중소기업의 금융 상태를 하나의 금융 디지털 트윈으로 통합하고, 17개 전문 AI가 분석·설계·실행을 협업하는 웹 금융 운영체제다. MVP는 범위를 축소하지 않으며 17개 서비스, 155개 하위 기능, 권한 단계, 샌드박스 실행과 감사로그를 모두 제공한다.

## 공통 기능 계약

모든 하위 기능은 다음 계약을 만족해야 한다.

1. 사용자가 서비스와 하위 기능을 선택하거나 자연어·음성으로 요청한다.
2. 금액, 기간, 위험 허용도와 기능별 추가 입력을 받는다.
3. 결정론적 계산 엔진이 금액·비율·시나리오를 산출한다.
4. 전문가 에이전트가 결과, 근거, 가정, 기준시각, 위험과 실행 순서를 설명한다.
5. 실행 요청은 L0~L4 권한, 1회·일간 한도, 기관 가용성, 사용자 재확인을 검사한다.
6. 허용된 요청은 샌드박스 어댑터에서 성공·실패·취소를 재현한다.
7. 입력, 계산 버전, 승인, 정책 판단과 결과를 D1 감사로그에 저장한다.

## 데이터 및 실행 구조

- 합성 금융계좌 16개: 은행, 예·적금, 국내·해외증권, 연금, ISA/CMA, 외화, 가상자산, 부동산, 자동차, 금·은, NFT, 주택·신용대출, 카드, 기업계좌
- 데이터 원천과 기준시각을 계좌·결과 단위로 표시
- D1 `action_requests`: 사용자, 멱등키, 서비스, 기능, 금액, 권한, 작업, 어댑터, 결과코드, 외부 참조번호 저장
- D1 `audit_logs`: 성공·실패·취소 결과와 상세근거를 사용자별로 저장하며 브라우저에서 직접 기록할 수 없음
- D1 `documents` + R2 `DOCUMENTS`: 문서 메타데이터·SHA-256·추출 신호와 최대 10MB 원문을 분리 저장
- 기관 어댑터: MVP에서는 `FIA Sandbox Adapter v1`; `InstitutionAdapter` 계약을 유지한 채 승인된 마이데이터·오픈뱅킹·금융기관 API로 교체
- 사용자 분리: 호스팅 인증 이메일을 내부 SHA-256 소유자 ID로 변환하고 모든 조회·쓰기에 적용

## 최종 완료 기준

- 155개 하위 기능 각각에 대표 시나리오와 고유 검증 케이스가 존재한다.
- 순자산·현금흐름·투자·재무설계·세금·보험·대출·소비·신용·위험·환율·기업금융의 기준 데이터 계산값이 기대값과 일치한다.
- 모든 실행은 승인 없이는 발생하지 않고, L4 고위험 작업은 매번 재확인한다.
- 성공·실패·취소·중복·한도초과가 감사로그에 누락 없이 기록된다.
- 데이터 원천, 기준시각, 계산 가정과 모델·규칙 버전을 결과에서 확인할 수 있다.
- 데스크톱·모바일에서 핵심 흐름을 키보드와 터치로 완료할 수 있다.
- 배포 URL, 기능명세, 실행·테스트 문서와 평가용 합성 데이터가 함께 제공된다.

## MVP와 상용화 경계

MVP의 155개 기능은 모두 입력→계산/추출→근거 결과→승인→샌드박스 실행 또는 저장 흐름으로 작동한다. 실제 계좌 조회, 송금, 주문, 보험·대출 신청, 세금 신고, 실시간 시세·뉴스는 자격과 계약을 갖춘 기관 API가 없으면 실행할 수 없으므로 동일한 어댑터 계약의 샌드박스 응답을 사용한다. 이 외부 연동은 기능 누락이 아니라 상용화 의존성이다.
