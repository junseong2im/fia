## 실제 API 통합검사

| 시나리오 | 기대 | 확인값 |
|---|---|---|
| L4 100만원 자동 송금 | 성공·감사기록 | HTTP 201 / `SANDBOX_COMPLETED` |
| L4 600만원 자동 송금 | 1회 한도 차단·감사기록 | HTTP 201 / `POLICY_LIMIT_EXCEEDED` |
| 최종 승인 취소 | 금액 이동 없음·취소 기록 | HTTP 201 / `USER_CANCELLED` |
| 동일 멱등키 재요청 | 기존 결과 반환·재실행 없음 | HTTP 200 / `duplicate: true` |
| 권한 위조 | 서버 카탈로그 불일치 차단 | HTTP 400 |
| 다른 Origin의 액션 | 교차 출처 차단 | HTTP 403 |
| 감사로그 직접 POST | 쓰기 경로 폐쇄 | HTTP 405 |
| 정상 TXT 계약서 | R2 저장·D1 메타데이터·본문 분석 | HTTP 201 |
| TXT를 PDF로 위장 | 파일 시그니처 불일치 차단 | HTTP 415 |

## 수동 QA 체크리스트

- 1440px, 1024px, 768px, 390px에서 홈·서비스·계좌·기록·권한 화면의 잘림 확인
- 키보드만으로 메뉴, 기능 칩, 입력, 결과, 승인·취소 완료
- `/` 검색 포커스, `Escape` 모달·모바일 메뉴 닫기 확인
- 음성 API 지원·미지원 브라우저의 안내 상태 확인
- PDF의 스캔 이미지 문서는 `OCR 필요` 상태로 표시되는지 확인
- 스크린리더에서 분석·문서 처리·토스트 상태가 읽히는지 확인

실제 브라우저 시각 회귀와 스크린리더 수동 검사는 배포 URL에서 별도로 수행해야 한다.
